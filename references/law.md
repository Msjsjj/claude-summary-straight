# 法学分支规则（law）

> 外层宏观骨架（复习定位 / 主线主题区 / 考试考向与例题 / 易错点 / 考前速记）见 SKILL.md 步骤 4，三分支共用。本文件仅定义 law 分支**主线主题的内部节格式**与专属规则。

## 总原则

> 本分支所有引号统一用大陆弯引号 ""''，禁止「」『』与 ASCII 直引号。

法学笔记的覆盖标准沿用"不读原材料也能理解并通过"。法学的"理解"重心是：记住一个法律制度的要件、法律效果、例外，以及制度之间的区分，而非大段叙述立法背景。每个制度优先拆成"是什么—要件—法律效果—例外或与相邻制度的区分"四层。

## 格式选择规则（law）

拿到一个子话题，先判断法学信息类型再选格式，不要默认写散文：

| 法学信息类型 | 强制格式 | 典型场景 |
|---|---|---|
| 构成要件、成立条件 | 编号清单，每个要件一行，要件后接一句认定要点 | 商事登记成立要件、表见代理构成 |
| 法定构成、必备记载事项 | 编号清单逐项列出，每项一句说明；有法定例外的另列 | 企业名称四段式构成、票据必须记载事项、公司章程必备条款、登记事项 |
| 两个相邻制度辨析 | 对比表，行=区分维度，列=被比较对象，表后必须一句话点出区分实益 | 子公司与分公司、撤销与解除、无效与可撤销 |
| 三个以上同类类型 | 每类型独立子节，结构一致：定义→特征→设立或责任要点→与其他类型一句话区分 | 各类公司、各类合伙、各类票据 |
| 程序、流程 | 步骤表，列=阶段／主体／行为／法律后果；带分支的流程用文字编号加箭头 | 公司设立、破产程序、证券发行核准 |
| 法律效果、责任承担 | 分情形列出：条件→后果；条件不同后果不同的用小标题分情形 | 违约责任、瑕疵担保、越权代表效力 |
| 概念定义 | 先一句精确定义，再拆关键词，每个关键词一句解释为何写进定义 | 商行为、商事主体、商号 |
| 法条要点 | 写"法律名称＋第 X 条"，正文转述要点，不照抄整条 | 公司法关于法定代表人的规定 |

解释性表格仍须配文字说明（表前引导＋表后提炼区分实益），沿用 SKILL.md 共享硬约束。

区分"法定构成／组成部分"与"成立要件"：前者是"由哪几部分组成／必须记载什么"，后者是"怎样才算合法成立"，二者是不同考点，不能以其一顶替另一。许多商法概念（企业名称、票据、公司章程、登记事项）两者都有，须分别成节。

## 主线主题内部节模板（law 专用）

每个 `## 主线主题` 内部，按以下节结构展开（顺序按实际内容定）：

```
X.1 [概念／定义节]
（一句精确定义 + 关键词拆解）

X.2 [构成要件节]
（编号清单，每要件一行 + 认定要点）

X.3 [法律效果／责任节]
（分情形：条件→后果）

X.4 [相邻制度辨析节]
（对比表 + 表后区分实益一句）
```

三个以上同类类型（各类公司、各类合伙等）用独立 `####` 子节，结构一致：定义→特征→设立或责任要点→与其他类型一句话区分。

## 案例分析题骨架（本科级，写进"考试考向与例题"板块）

难度卡在本科线：要件不超过三到四个；涵摄写到"事实对要件"即可；不讨论举证责任、不要求请求权基础链条；结论给明确的是／否 + 一句法律后果。骨架直接复用正文已写好的要件清单和对比表，不另起炉灶。

**两种问法：**

效力或责任判断题（如"是否构成表见代理、是否担责"）：

```
**例题** · 案例分析题 · [问的是什么]
[定性] 本案争议是……，涉及的法律制度是……
[规则] 该制度成立要件：①…… ②…… ③……
[涵摄] 逐一对照本案事实：要件①是否满足——事实是……；要件②……；要件③……
[结论] 因此……成立／不成立，……应／不应承担……责任
```

区分或选择题（如"认定为分公司还是子公司、无效还是可撤销"）：

```
**例题** · 案例分析题 · [二者择一]
[争点] 本案需在 A 与 B 之间定性
[标准] A 与 B 的核心区分在……
[归入] 本案事实符合……的特征，因为……
[结论] 应认定为……，法律后果是……
```

## 法学专属反面教材（不要重复）

1. **把构成要件写成大段散文**——要件必须拆成可勾选编号清单，散文里埋着要件等于没拆
2. **只背结论不记区分**——相邻概念辨析是高频考点，只写"甲是什么乙是什么"不写"甲乙怎么分"等于没复习到考点
3. **用修订前旧法当现行法**——商法尤其致命，公司法 2023 年底大改，无源检索极易抓到旧内容
4. **臆造条文号**——记不准的条号宁可只写法律名称＋要点，不写第几条
5. **只讲是什么不讲法律效果**——考点常在"成立后什么效果、不成立什么后果"，光定义是半截笔记
6. **遗漏法定构成／必备记载事项**——把"构成"用"成立要件"顶替（如商业名称只写命名规则，漏掉行政区划＋字号＋行业＋组织形式四段式构成）
7. **用过时法条**——只查二手资料、或抓到旧版法规当现行法（企业名称 1991 旧版、公司法 2023 修订前等），未核验是否现行有效

## 无源模式法条策略（主动检索＋核验现行有效，仅 1B 模式叠加）

- 凡某知识点以某部法律／法规为依据（构成、要件、效力、记载事项等），必须主动 web_search 该法规原文条款，不能只依赖知乎、智库等二手复习资料——二手资料常遗漏行政法规层面的细节（例：企业名称四段式构成出自《企业名称登记管理规定》，商法教材总结里常不突出）。
- 检索法条必须核验现行有效版本：确认施行日期、是否已被修订或废止。搜索结果常同时返回旧版与新版（企业名称登记规定有 1991 旧版与 2021 现行版，两版名称构成表述不同；公司法有 2023 修订前后版本），务必采用现行有效版，并在笔记中标注法规名称与已确认的施行／修订时间。
- 条文号在确认来源且核验现行有效后即可写出；只有确实无法检索确认到具体条号时，才退而写"XX 法对此有规定"＋要点转述。
