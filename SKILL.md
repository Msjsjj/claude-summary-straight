# Obsidian 课程章节复习笔记生成

把课程资料整理成结构化、可直接复习的章节笔记，写入 Obsidian Vault 的 `Summary-straight/[课程名]/` 文件夹下。

**Vault 路径**：用户 Obsidian Vault 下的 `Summary-straight\` 子文件夹（具体路径在步骤 2 动态确定）

## 核心约束

输出物 **必须直接写入 Vault 文件夹**——不在对话里输出完整笔记内容。

笔记面向 **期末考试复习**：要"自包含"，可以脱离源资料独立复习；机制要详写，事实清单要完整；不堆砌"重要度"标签让用户自己判断重点。

**全面覆盖原则（最高优先级约束）**：笔记的覆盖标准是——**一个从没上过这门课的人，仅凭这套笔记就能理解并通过考试**。这意味着：

- 概念性框架（状态图、流程图、"为什么需要这个机制"）和算法计算内容同等重要，不可因"看起来基础"而跳过
- 读 PDF/PPT 时，所有页面的内容都必须有去留判断，不允许因为"不像考点"或"是导论铺垫"就自动略过
- 跳过任何内容前，必须能回答：「用户不读原材料，能从笔记里理解这部分知识吗？」——不能则不跳过

**反面示例（本 skill 已发生的错误）**：进程调度课件第 15 页的六状态图（new/ready/running/blocked/ready-suspend/blocked-suspend）及三类调度与状态转移的对应关系，因"看起来是背景铺垫"被遗漏。但这恰恰是理解三类调度为何存在的根基，缺失后其余内容变成死记硬背。

笔记有 **两种输入模式**：
- **有源资料模式**：用户上传 PPT / PDF / Word。基于资料内容写。
- **无源资料模式**：用户只告诉课程名+章节，没传资料。基于外部检索写，更严格的真实性约束（见步骤 1 无源分支与步骤 3 强化）。

---

## 工作流（6 步）

### 步骤 1：资料分析与章节归属判断

**判断当前是哪种输入模式**，分支处理。

#### 1A：有源资料模式

阅读上传的资料，识别：

- 资料覆盖的章节（可能是单章节，也可能横跨多章）
- 资料属于哪门课（如"应用密码学"、"操作系统"等）

判断依据：资料目录、章节标题层级、关键概念词。

**单章节** → 直接进步骤 2。

**多章节** → 反问用户："这份资料覆盖了 [章节 A、章节 B、章节 C]，你想：(1) 只做某一章 (2) 全部都做（每章独立一篇笔记） (3) 合并成一篇章节总览。" 等用户回复再继续。**不要擅自决定**。

#### 1B：无源资料模式

用户没上传文件、但要求写复习笔记。**必须先问清三件事**才能开始：

1. **教材**（书名+作者，例如"谢希仁《计算机网络》"、"Stallings《密码学与网络安全》"）
2. **版本**（第几版，影响章节编号和内容增减）
3. **章节**（具体到节，例如"4.5 路由算法"、"Chapter 5 古典密码"）

**不要从历史对话推断**。不同课程教材差异很大，瞎猜会导致笔记内容与用户实际学的内容偏差。

得到这三项后才能进步骤 2。

### 步骤 2：Obsidian 文件夹定位

在 `Vault/Summary-straight/` 下检查是否有对应课程的文件夹（用 PowerShell `Get-ChildItem`）。先确认用户的 Vault 根路径（可查看 Claude Code 的工作目录，或让用户告知），然后：

```powershell
Get-ChildItem "<用户Vault根路径>\Summary-straight" -Directory
```

**找到唯一匹配** → 记下完整路径，顺带扫描课程文件夹下所有现有笔记文件名（去掉扩展名后的词干列表，用作步骤 4 的 WikiLink 目标）：

```powershell
Get-ChildItem "D:\...\Summary-straight\[课程名]" -Recurse -Filter "*.md" | Select-Object -ExpandProperty BaseName
```

**找到多个候选** → 列出让用户选。

**完全没找到** → 反问用户："Vault/Summary-straight/ 下没找到 [课程名] 文件夹。你要：(1) 我新建一个 (2) 用其他已存在的文件夹（请告诉我名字）"。

### 步骤 3：重点信号收集 + 例题搜集

本步骤有**两个并行任务**，缺一不可：① 确认考试重点；② 搜集例题。

#### 3A：有源资料模式

**任务①——确认重点：**
如果用户已明确说明重点（"老师强调了 X"、"这章要会画图"），用户提示**优先级最高**，跳过重点搜索。
否则，**必须**用 `web_search` 查可引用的外部信号验证重点判断，至少 2 次：

- `[课程名] 考研真题 [章节]`
- `[课程名] [章节] 期末考试重点`

**任务②——搜集例题（不可跳过）：**

先检查用户上传的源资料中是否含有例题/习题/真题：
- **资料有例题** → 优先采用资料中的例题，可补充 1-2 次 WebSearch 扩充题型覆盖
- **资料无例题** → 必须 WebSearch，至少 2 次，搜集各题型例题

WebSearch 搜索建议：
- `[课程名] [章节] 期末考试题 例题 答案`
- `[课程名] [章节] 真题解析 选择题 简答题`
- `[课程名] [章节] 填空题 判断题 答案`

#### 3B：无源资料模式（强化版）

**任务①——确认重点（强制）：** 至少 2 次 web_search，外部检索是唯一的事实锚点：

- `[教材+版本] [章节] 重点`
- `[课程名] [章节] 知识点 总结`
- 关键概念名 + "定义" 或 "公式"（核对术语和写法）

**任务②——搜集例题（强制）：** 至少 2 次 web_search，无源资料时 WebSearch 是例题的唯一来源：

- `[课程名] [章节] 期末考试题 例题 答案`
- `[课程名] [章节] 考研真题 解析`
- `[课程名] [章节] 选择题 简答题 填空题`

**硬规则**（两种模式共用）：每一条"这是重点"的判断必须能指向具体来源。无来源就标"我的推断"，不写"重点 / 必考 / ★"。

### 步骤 4：笔记生成

#### 文件命名格式

`[主题]期末备考笔记：[章节名].md`（冒号用全角 `：`，Windows 文件名合法）

例如：`古典密码期末备考笔记：替代密码与置换密码.md`

#### 文件头 frontmatter（每篇必须）

```yaml
---
date: YYYY-MM-DD
tags: [课程名, 复习笔记]
course: 课程名          # 与文件夹名一致，供 Dataview 查询
chapter: 7              # 章节编号（纯数字），便于排序
source: 路径/文件.pdf   # 有源资料时填写；无源资料时省略
status: draft           # draft | reviewed | exam-ready
---
```

`status` 字段生命周期：写完填 `draft`，自己过了一遍填 `reviewed`，考前确认填 `exam-ready`。

#### 格式选择规则

拿到一个子话题时，先判断信息类型再选格式——不要默认写段落散文：

| 信息类型 | 强制格式 | 典型场景 |
|---------|---------|---------|
| 算法参数/规格 | 表格（行=参数，列=取值/备注）| 分组长度、密钥长度、轮数、标准 |
| 算法流程（有分支/框图） | ASCII 流程图（代码块，用 `↓` 和 `┌─┐` 框）| 加解密总流程、密钥扩展流程 |
| 算法流程（线性，无分支） | 编号列表（`①②③…`）+ 行内 LaTeX，每步一行 | SHA-1 整体流程、填充→分组→迭代→输出 |
| 步骤性过程 | 步骤表（序号｜操作｜输入→输出位宽｜作用）| 轮函数各步骤、变换步骤 |
| 并列对比两个概念 | 对比表（行=特性维度，列=被比较对象）| 混淆 vs 扩散、Feistel vs SP 网络 |
| N 个同类对象（N ≥ 3）| 每对象独立 `####` 子节，内部结构完全一致 | 五种工作模式、四个 AES 变换 |
| 特性/约束/限制 | ✅/❌/⚠️ bullet（✅ 优点，❌ 缺点/限制，⚠️ 注意）| 并行性、错误传播、IV 要求 |
| 密码协议交互步骤（单方视角） | 编号列表（`①` `②`…）+ 行内 LaTeX，每步一行 | KDC 协议、公钥分配步骤 |
| 密码协议交互图（双方对话） | 三列表格（Alice \| 箭头 \| Bob），箭头用 `$\xrightarrow{\quad s \quad}$` / `$\xleftarrow{\quad s \quad}$` | DH 交换、STS 协议 |
| 网络拓扑 / 架构图（多实体方框、跨边界连线、含中文标注） | Mermaid `flowchart LR` + `subgraph`，中文标注写在 `subgraph` 标题里；节点标签内换行用 `<br/>`，不用 `\n` | eBGP/iBGP 分工图、AS 拓扑、协议分层框图 |

**N 个同类对象的子节模板**（固定三段，顺序不变）：
1. 公式 / 操作步骤
2. 特性（✅/❌/⚠️ bullet）
3. 适用场景一句话

#### 骨架模板

```
## 一、复习定位
（一段话总述这章解决什么问题；列出本章主线和优先级排序）

## 二、[主线主题 A]
### 2.1 [参数/规格节]
（用参数表格）

### 2.2 [流程/步骤节]
（用 ASCII 流程图或步骤表，按格式选择规则判断）

### 2.3 [N 个同类对象节]
（每对象一个 #### 子节，内部结构完全一致）

## 三、[主线主题 B]
...

[更多主题章节...]

## [N]. 考试考向与例题（标准板块，必写）

### N.1 考法分析
（按题型列出高频考点与难度：选择题 / 填空题 / 简答题 / 画图题 / 分析题）

### N.2 选择题
（2-4 道带解析，覆盖本章核心易混点）

### N.3 填空题
（2-4 道，覆盖定义、步骤、公式等直接考察型内容）

### N.4 简答题
（2-3 道带答题要点，覆盖"比较 A 和 B"、"说明步骤"等高频简答类型）

### N.5 画图 / 分析题（如本章有此题型则写，否则省略）
（1-2 道场景分析或画图题，带完整解题过程）

## [N+1]. 易错点（收尾板块 1，可选）

> [!WARNING]
> **易错点标题**
> - 易混淆的概念或计算步骤……

## [N+2]. 考前速记（收尾板块 2，可选）

> [!TIP]
> **速记标题**
> - 一句话核心……
```

#### 硬约束（违反任何一条要立即修复）

- **章节编号必须连续**——一/二/三/四，不能跳号
- **"考试考向与例题"是标准必写板块**——不计入收尾板块数量；例题来源优先级：用户资料中的例题 > WebSearch 搜集的例题
- **收尾板块最多 2 个**——指"考试考向与例题"**之后**的附加板块，典型组合：易错点 + 考前速记。不要罗列 5 个同义板块
- **不写工作流元规则进笔记正文**
- **emoji 仅限白名单**：✅/❌/⚠️ 只能用于 bullet 特性列表（✅ 优点，❌ 缺点/限制，⚠️ 注意）；标题、散文正文、表格中禁止使用 emoji
- **不写 "★ / 必考 / 重点"**——用户自己判断重点
- **中文标点用大陆标准弯引号 ""''**，括号、冒号、分号、感叹号、问号用全角
- **解释"为什么"而不只说"是什么"**
- **解释性表格必须配文字说明**——凡是用于对比概念、列举规格、展示规律的表格，必须在表格前或表格后附加简短文字，说明"怎么读这张表""关键规律是什么""如何选择"等。纯数据表格（例题输入数据、计算结果）不受此约束。典型做法：表前一句引导（"常用以下五个指标："）+ 表后 1-3 句提炼规律（"有饥饿风险的…；无饥饿的…"）
- **同类对象格式一致**——同类型 N ≥ 3 个对象（工作模式、变换、算法对比等）必须全部用相同格式；不允许同类对象前几个写散文、后几个写表格
- **代码块只用于 ASCII 流程图和伪代码**——协议步骤（①②③…）和加解密公式一律用行内 `$...$` LaTeX，不得放进代码块（代码块内 LaTeX 不渲染，`E_{K_A}[...]` 将原样显示为纯文本）

#### WikiLink 联结

步骤 2 扫出的同课程笔记词干列表，在正文生成时与正在写的内容匹配：凡提到与现有笔记同名的章节/概念，用 `[[词干]]` 替换首次出现的纯文本。不要过度链接——每篇笔记内同一词条只链一次（首次出现处），且仅链接确实存在的词干（不猜测未来笔记）。

#### 加粗原则

`**加粗**` 用于核心概念第一次出现的关键名词或关键判断的核心词；`*斜体*` 用于英文术语或需要对比意味的措辞。加粗克制使用，避免整句加粗（加粗一整句话等于没有加粗）和整行列表项加粗（`- **整行内容**`）。

#### 无源资料模式的额外写作约束

仅在 1B 模式下生效：

- **不确定的细节用引用块（`>` 开头）或代码字体（`` ` `` 包裹）显著区分**。
- **不写具体页码 / 题号**。这种细节最容易瞎编。
- **笔记末尾追加"对照清单"**。3-8 条让用户拿这份笔记跟教材对一遍。**用编号列表 `1. 2. 3.` 写**，每条独立段落，每条开头用加粗短词起头（如"**单表替代密码的具体子类**"）。

### 步骤 5：写入 Obsidian

**写入前查重**：

```powershell
Test-Path "D:\...\Summary-straight\[课程名]\[文件名].md"
```

如发现同名文件，询问用户："已存在 [文件名]，要 (1) 覆盖 (2) 新建独立文件（加序号）(3) 取消"。

**写入前运行 pre_check**（捕捉非法 LaTeX 行内公式）：

1. 把草稿写到临时文件（Windows：`$env:TEMP\obsidian_draft.md`；macOS/Linux：`/tmp/obsidian_draft.md`）
2. 运行 `python scripts/pre_check.py <临时文件路径>`
3. 若报告 `inline_formula_validity` 失败，定位并修复对应公式（`$x = $` 等号一侧空是最常见问题）
4. 通过后执行写入

**PDF 读取**：`Read` 工具不支持 PDF，需用 pypdf：
```python
import pypdf
r = pypdf.PdfReader(r'路径\文件.pdf')
text = ''.join([p.extract_text() for p in r.pages])
```

**写入**：用 `Write` 工具写入 `Vault/Summary-straight/[课程名]/[文件名].md`

**token 节省硬约束**：
- **禁止 `cat` 全文输出草稿**——写入前、修改后验证，都禁止把草稿完整 cat 出来
- **禁止写入后重读全文验证**——用 self_check 脚本验证即可
- **禁止重复 print 全文**

### 步骤 6：写后自检

对已写入的文件**依次运行两个脚本**，两个都通过才算完成：

```powershell
python scripts/self_check.py "D:\...\Summary-straight\[课程名]\[文件名].md"
python scripts/post_check.py "D:\...\Summary-straight\[课程名]\[文件名].md"
```

**self_check 检查项：**
1. 章节编号连续性
2. 工作流元规则泄漏
3. 收尾板块数量（≤ 2）

**post_check 检查项：**
1. `example_present`——笔记含 `**例题**` 加粗格式（注意：写 `**题目**` 不算）
2. `external_urls`——笔记正文含外部链接（复习定位节引用 web_search 来源 URL）
3. `frontmatter`——包含 date / course / chapter / status
4. `wikilink`——含 `[[...]]` 联结同课程其他笔记

**任一项失败** → 定位问题、直接用 Edit 修改 Vault 文件、重新跑对应脚本。

**全部通过** → 把文件路径给用户（可在 Obsidian 中直接打开）。

## Obsidian 公式规则

Obsidian 使用 KaTeX 渲染，基本规则：

- **行内公式**：`$...$` — 反斜杠命令（`\frac`、`\alpha` 等）可以直接用，无需改为块公式
- **块公式**：`$$...$$` — 复杂多行公式或需要居中展示时用
- **列表项里可以用公式**（Obsidian 无此限制）
- **与中文紧贴也能渲染**（Obsidian 无须强制加空格）

**唯一需要避免的**：写出不合法的 LaTeX 表达式，如 `$x = $`（等号一侧空）、`$+$`（只有运算符）等。

## 反面教材（不要重复这些错误）

1. **工作流元规则混入笔记正文**——比如笔记里突然出现"公式书写规范"小节
2. **章节编号跳号**——"一/二/四/五"漏掉"三"
3. **主线和支线平铺**——"古典密码 = 替代 + 置换"是主线，"频率分析"是攻击手段应归到"替代密码"之下
4. **收尾板块过多**——同时出现"高频题型 / 最小复习清单 / 易错点 / 考前速记 / 复习顺序"五个同义板块
5. **例题缺失或分布不均**——"考试考向与例题"是必写标准板块；例题来源不足时必须 WebSearch 补充，不能跳过或只写 1-2 道选择题了事
6. **"备考点"列表是正文复述**——"备考点：凯撒密码是移位密码的特例"就是上文已经说过的话
7. **只说"是什么"不说"为什么"**
8. **缺少跨章节联系**
9. **emoji 用错场景**——✅/❌/⚠️ 只能在 bullet 特性列表里用；标题、散文、表格里出现 emoji 一律违规
10. **过度小标签**——大量"加密：解密：条件：备考点："让阅读节奏破碎
11. **修复时重写整篇**——pre_check / self_check / post_check 报错时默认用 Edit 精确修复，重写消耗大量 token
13. **只跑 self_check 不跑 post_check**——两个脚本检查不同维度，Step 6 必须两个都跑；post_check 检查例题、外部链接、WikiLink，self_check 检查章节编号和元规则，缺一不可
14. **把概念框架当"背景"跳过**——状态图、流程图、"为什么需要这个机制"等内容不是可选铺垫，是让算法知识有根基的必要内容；判断标准：不读原材料的人能否从笔记里理解？不能则不跳过
12. **协议步骤放进代码块**——`E_{K_A}[K_S || ID_A]` 在代码块里是纯文本；正确写法：`① $A \to B$：$E_{K_A}[K_S \| ID_A]$`
14. **表格孤立无文字**——解释性表格（决策模式、性能指标、算法对比等）只写表格、不加任何解释，没学过的人看了会云里雾里；正确做法：表前加一句引导，表后提炼 1-3 句关键规律或选择思路
15. **拓扑图用 ASCII 画框**——含中文标注的网络拓扑图，中文字符双倍宽度与 ASCII 边框天然错位（"全互联"三字会把边框撑出格），越修越乱；正确做法：一律用 Mermaid `flowchart LR` + `subgraph`，中文标注写在 `subgraph` 标题里，Obsidian 原生渲染，无需手动对齐
16. **图文不一致**——画图时照搬原材料标注，未用上下文文字校对图的逻辑，导致图与正文描述矛盾（如热土豆路由图中"AS间跳数更多"标在了错误的一侧）；正确做法：画完图后对照周围文字逐项核查"哪侧更多/更少、哪个被选/不被选"，确保图文逻辑完全一致
